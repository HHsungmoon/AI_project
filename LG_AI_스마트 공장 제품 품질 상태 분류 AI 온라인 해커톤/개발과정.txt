스마트 공장 제품 품질 상태 분류 AI 온라인 해커톤(2023-2-28)

-데이터 분석
1. 데이터셋 type 
	A,T,O 세가지 종류의 제품이 있다. O 타입의 데이터 갯수가 6개 밖에 없다.
	A와 T,O의 데이터 특성(colums)이 매우 다르다. 
	target이 0,1,2 인데 품질 점수가 0.53을 기준으로 0.1 정도의 작은 범위이기 때문에 이를 기반으로 분류 모델을 만들기는 어렵다 판단.

-프로그래밍 과정
1. Z-score 데이터 정규화 진행, 이상치 제거 고려
2. 문자열colum을 수치화 진행
3. dataloader에서 batch size를 16,32,64 로 설정
4. MLP_typeA 모델과 MLP_typeTO 모델 생성
	-layer: 2876(input)->512(h1)->64(h2)->3(output) 
	-layer가 4,5로 깊어져도 성능향상이 어려움
	-hidden layer의 노드수가 많아 dropout(0.4~0.2)을 적용해 과적합을 완화
	-sigmoid->relu->selu 활성화 함수를 바꿨는데 성능향상. selu의 성능이 가장 좋았다.
	
5. 손실함수 CrossEntropyLoss
	-훈련 데이터의 target 불균등이 크다(0,1,2 -> 88:407:103)
	-따라서 손실함수의 weight를 1:4:1로 설정해 주었다. 
6. optimizer는 Adam을 사용 
	-learning rate는 0.001로 설정
	-weight_decay를 0으로 설정. 0.001부터 다양하게 시도해 봤지만 성능이 좋지 않았다.
7. train과정에서 최적의 파라미터를 저장한 뒤 사용

-결과- acc = 0.592
1. 훈련 데이터 부족과 편향으로 인해 과적합 문제가 발생한 것 같다.
2. train_acc는 안정적으로 학습되나 test_data의 예측이 제대로 되지 않았다. 
3. 훈련세트에 target class가 1이 많은데 test예측결과에도 1이 많다.

-후기-
첫 해커톤 대회 참여였는데 생각보다 어려웠다. 최종 순위는 상위30% 정도. 
acc 0.65 이상을 목표로 했지만 제출횟수 제한 때문에 파라미터 조정을 많이 하지 못해 아쉬웠다. 
지금까지 공부하며 사용했던 잘 정돈된 dataset에 비해 데이터 정규화 부터 진행해야 해서 어려움이 있었다. 
앞으로 이런 대회에 참여하며 다양한 경험과 실력을 키워보고 싶다.
